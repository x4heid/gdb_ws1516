\subsection*{c) Unsicherheit}
Beim Löschen eines Eintrags aus BenutzerIn gibt es 2 unterschiedliche mögliche Effekte auf die Einträge von Rubrikzuordnung.\\
Das Löschen könnte auf direktem Weg kaskadieren oder per indirektem Weg über Rubriken verboten werden und so unterschiedliche Ergebnisse erzielt werden, was das Schema unsicher macht.

\subsection*{d) Vorkehrungen}

\section*{Aufgabe 2: Änderbarkeit von Sichten}

\subsection*{Sichterstellung}
\subsubsection*{i) Enterprise-Crew}
\begin{verbatim}
CREATE VIEW EnterpriseCrew AS
    SELECT b.BNr, b.Name, b.Rang
    FROM Raumschiffe r, Besatzungsmitglieder b
    WHERE b.Schiff = r.RNr
    AND r.Name = 'Enterprise';
		
Die Sicht ist nicht änderbar, da hier Tabellen verbunden werden.
\end{verbatim}

\subsubsection*{ii) Captains}
\begin{verbatim}
CREATE VIEW Captains AS
    SELECT Name
    FROM Besatzungsmitglieder
    WHERE Rang = 'Captain';
		
Auch hier ist die Sicht nicht änderbar, doch diesmal aus dem Grund,
dass sie nicht den Primärschlüssel der referenzierten Besatzungsmitglieder-
Relation enthält.
\end{verbatim}

\subsubsection*{iii) WarpFed}
\begin{verbatim}
CREATE VIEW WarpFed AS
    SELECT RNr, Fraktion, Baujahr
    FROM Raumschiffe
    WHERE Geschwindigkeit >= 1
    AND Fraktion = 'Förderation';

Diesmal ist die Sicht änderbar.
\end{verbatim}
